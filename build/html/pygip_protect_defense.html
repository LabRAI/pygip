pygip.protect.defense¶

class pygip.protect.Defense.Defense(dataset, attack_node_fraction)[source]

Bases: object

A base class for defense (or watermark) operations. In this example, it is mainly used to merge watermark graphs with existing datasets or perform subsequent model extraction attacks.

A GraphSAGE model implemented with PyG’s SAGEConv module.

It consists of two SAGEConv layers: - The first layer projects features to ‘hidden_channels’, - The second layer outputs ‘out_channels’.

class pygip.protect.Defense.WatermarkGraph(n, num_features, num_classes, pr=0.1, pg=0, device=device(type='cpu'))[source]

Bases: object

A class for generating a watermark graph.

This class first uses the ‘erdos_renyi_graph’ function to randomly generate edges, then randomly generates node features and labels, and finally creates a DGLGraph with the generated information.

class pygip.protect.Defense.Watermark_sage(dataset, attack_node_fraction, wm_node=50, pr=0.1, pg=0, device=device(type='cpu'))[source]

Bases: Defense

Inherits from Defense. Uses GraphSAGE to train on both the original and watermark graphs, ultimately embedding a watermark into the trained model (or graph).

attack()[source]

Main attack (or watermark injection) process: 1. Generate the watermark graph (data_wm). 2. Pre-train on the original graph via GraphSAGE. 3. Optionally fine-tune on the watermark graph.

class pygip.protect.Defense.graph_to_dataset(graph, attack_node_fraction, name=None)[source]

Bases: object

A utility class to wrap a DGLGraph into a dataset-like structure.

Main responsibilities: 1. Extract basic graph information such as the number of nodes, features, labels, and attack nodes. 2. Retrieve and process node features, labels, as well as train/test masks. 3. Ensure that all tensors are on the specified device (device).
